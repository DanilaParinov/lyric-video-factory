# lyric-video-factory — Project Context for Claude

## Описание проекта
Сервис для автоматической генерации lyric video с веб-интерфейсом.

Пользователь предоставляет:
- аудиофайл
- текст песни
- темплейт с таймингами (JSON)
- набор фоновых видеофайлов
- желаемое количество вариантов

Сервис генерирует N уникальных lyric video, каждый раз используя случайную подборку фоновых клипов.

## Стек
- **Go 1.26.4** — основная кодовая база, бизнес-логика
- **FFmpeg** — вызывается через `os/exec` (без сторонних обёрток)
- **Go net/http** — HTTP сервер, отдаёт REST API и статику
- **HTML + JS** — веб-интерфейс (без тяжёлых фреймворков)
- Финальная сборка — единый `.exe` со встроенной статикой через `embed`

## План разработки

### Фаза 1 — Прототип FFmpeg в Go ← текущая фаза
- Вызов FFmpeg через `os/exec`
- Hardcode-параметры: один фоновый клип + одна строка текста → видеофайл
- Цель: убедиться, что связка Go → FFmpeg работает

### Фаза 2 — Формат темплейта
- JSON-файл с таймингами, позицией текста, стилями
- Парсинг в Go-структуры, валидация входных данных

### Фаза 3 — Рандомизация и batch-генерация
- Нарезка фоновых видео на сегменты под тайминги
- Случайная подборка фрагментов
- Генерация N вариантов параллельно (горутины)

### Фаза 4 — HTTP сервер + REST API
- Эндпоинты: загрузка файлов, запуск генерации, статус задачи, скачивание результатов
- Отдача статики (веб-интерфейс)

### Фаза 5 — Веб-интерфейс
- Загрузка файлов через браузер
- Запуск генерации и отображение прогресса
- Список готовых видео с возможностью скачать

### Фаза 6 — Полировка
- Обработка ошибок (FFmpeg не найден, кривой темплейт и т.д.)
- README с примерами использования
- Примеры темплейтов из коробки
- Сборка в единый бинарник (`go:embed` для статики)

## Соглашения
- Язык комментариев и коммитов: русский
- FFmpeg вызывается только через `os/exec`, без библиотек-обёрток
- Горутины используются для параллельной генерации вариантов
